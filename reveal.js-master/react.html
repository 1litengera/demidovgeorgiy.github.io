Где используется React?

React используют для создания интерактивных пользовательских интерфейсов в различных проектах:

Создание функциональных интерактивных веб-интерфейсов. При работе с такими интерфейсами не нужно постоянно обновлять страницу.
Быстрая и удобная реализация отдельных компонентов и страниц целиком. Элементы в React легко использовать повторно.
Лёгкая разработка сложных программных структур. Разработчики описывают сложные процессы в коде с помощью инструментария React.
Разработка одностраничных и многостраничных приложений (SPA и PWA). Это приложения, которые функционируют как программы и веб-сервисы и имеют соответствующий интерфейс.
Работа с серверной частью сайта или разработка мобильных приложений. В таких случаях React используют совместно с инструментами, адаптирующими веб-технологии под другие цели.
Доработка новой функциональности с любым изначальным стеком технологий. Библиотека не зависит от остального инструментария и будет хорошо работать, на чём бы ни был написан код.

История React

2011год — рождение React, начало как внутреннего проекта в Facebook. Инженер Джордан Уолке создал раннюю версию React под названием FaxJS, которая позже стала React
2013 год — React был представлен на конференции JSConf US, что стало началом его широкого распространения.
2014 год — React начал расширяться, был представлен текстовый редактор Atom и плагин React Hot Loader.
2015 год — было объявлено о выходе React Native, сначала для iOS, затем для Android.
2016 год — в Сан-Франциско прошла конференция React.js, где был представлен Draft.js.
2017 год — Facebook запустил React Fiber, новый набор алгоритмов стал основой для будущих улучшений библиотеки React.
2018 год — на конференции в Исландии было представлено Beyond React 16 и выпущена версия React 16.3.0.
2019 год — были выпущены новые React DevTools
2020 год — были выпущены серверные компоненты React с нулевым размером пакета, а также версия React 17.0.
25 апреля 2024 года была выпущена бета-версия React 19, которая представляет новые функции, такие как действия по обработке изменений данных и обновлений состояния, улучшения в рендеринге на стороне сервера и многое другое.

Отличия от других библиотек и фреймворков

Цель использования: React — библиотека для создания пользовательских интерфейсов, в то время как другие библиотеки могут иметь более широкие функции, например, предоставлять полноценную архитектуру приложения.
Декларативный код: в React описывают результат, а не способ его достижения.
Компонентная архитектура: основным строительным блоком в React является компонент, что позволяет разработчикам повторно использовать код, упрощает тестирование, масштабирование и поддержку приложения.
Виртуальный DOM: в отличие от реального DOM, виртуальный DOM полностью существует в памяти, разумно определяя, какие изменения нужно внести.
Модульность: благодаря модульности разработчики могут просто извлечь отдельный модуль и делегировать рабочую нагрузку новому члену команды.
Работа с приложениями реального времени: React отлично работает с событийно-ориентированной архитектурой, когда события отправляются через веб-сокеты.
Совместимость с TypeScript: React можно связать с TypeScript при помощи специальных настроек, но стандартно библиотека работает только с JavaScript.
Возможность миграции между версиями: React позволяет легко мигрировать между версиями, так как в библиотеке есть специальные скрипты, помогающие при обновлении

React: Redux и React hooks

Redux и React Hooks — это инструменты для управления состоянием в приложениях на базе React. Redux — это библиотека, которая реализует предсказуемое хранилище состояния приложений. React Hooks — это паттерн, который появился в React 16.8 и позволяет управлять состоянием функциональных компонентов. Redux и React hooks это дополнения.Redux подходит для управления глобальным состоянием и действиями, а React Hooks — для обработки состояния локальных компонентов.

Плюсы React

Большое сообщество. Множество разработчиков используют и поддерживают React, поэтому найти помощь или готовые решения задач легко.
Развитая экосистема. Проект поддерживает множество библиотек и других инструментов, которые можно использовать в разработке. Это упрощает процесс и позволяет решать большое количество задач.
Удобная отладка. Ошибок с React бывает меньше, чем при написании кода на «чистых» языках без библиотек. Это происходит, потому что многое реализуется автоматически, уменьшается влияние «человеческого фактора».
Эффективность. Благодаря виртуальному DOM библиотека экономит ресурсы. Чтобы изменить состояние элементов, не нужно полностью перезагружать всё DOM-дерево, тратя трафик пользователя и нагружая браузер.

Минусы React

Специфический синтаксис. В React используется JSX — расширение JavaScript, которое может казаться запутанным и сложным для освоения.
Сложности оптимизации для поисковиков. Так как при использовании React компоненты программируются на JavaScript, они обрабатываются медленнее. Какие-то данные могут оказаться совсем недоступными для поисковых роботов, что негативно сказывается на SEO.
Проект написанный на React может начать больше весить из-за множества библиотек

React — популярный и широко используемый инструмент для разработки пользовательского интерфейса. Он позволяет создавать динамические и отзывчивые веб-приложения.
